## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | System background / window material | Panel background and search field outer context |
| Secondary (30%) | `.ultraThinMaterial` / secondary surface material | Cards, search field surface, filter chip surface |
| Accent (10%) | System accent color | Selected filter chip fill/border, search focus ring, selected-card border, match highlight background |
| Destructive | System red | Not used in primary Phase 4 UI; reserved for destructive actions outside this contract |

Accent reserved for: active filter chip state, search field keyboard focus ring, selected card border, text/RTF search-match highlight background, clear-search button focus state. Accent must not be used for passive card backgrounds, default body text, or inactive filter chips.

## Interaction Contract

### Search and Filter Behavior

| Trigger | Action | Notes |
|---------|--------|-------|
| Panel opens | Focus search field immediately | Locked decision from CONTEXT.md |
| Panel reopens | Clear query, reset active filter to All, focus search field | Prescriptive phase default |
| User types in search field | Results update in real time | FR-4.2 |
| Query + active filter both present | Apply both in same pipeline | Locked decision |
| User selects filter chip | Replace current active filter | Mutual-exclusive single-layer model |
| User clears query | Preserve active filter | Recovery keeps filtering context |
| User switches filter while query exists | Keep query, rerun results | Query remains active |
| Search returns no items | Show no-results state, preserve current query and filter UI | Locked decision |
| Search field loses focus | Left/right arrow navigation returns to card navigation | Locked decision |

### Keyboard Navigation Priority

| Context | Left / Right Arrow | Enter | Escape |
|---------|--------------------|-------|--------|
| Search field focused | Move text cursor / select text range per macOS text editing | No-op for search submit; primary action remains card-level only when field is not actively editing | Close panel if field is empty and no modal state; otherwise standard text-field behavior may clear selection only if system default applies |
| Search field not focused + results available | Move to previous / next card | Run current card primary action | Close panel |
| Search field not focused + no results | No card navigation | Trigger Clear Search only if focus is on CTA and user activates it | Close panel |

### Selection Contract

- When results are available after query/filter application, selected card defaults to the first visible result.
- If the currently selected card disappears after filtering, selection resets to the first result.
- If zero results remain, selectedIndex becomes invalid/none and card navigation is suspended until results return.
- Search focus takes precedence over card selection focus on panel open.

### Search Matching Contract

| Field | Included in matching | Source |
|-------|----------------------|--------|
| `content` | yes | FR-4.1 + user confirmation baseline |
| File item filename | yes, derived from file content/path for `.file` items | RESEARCH.md recommendation |
| Source application name | no | user confirmation |
| Image binary metadata | no | default exclusion |

### Highlight Contract

| Item Type | Highlight | Notes |
|-----------|-----------|-------|
| Text | yes | Highlight matched substrings in preview text |
| RTF | yes | Treat as text for filtering and preview highlighting |
| File | no | Filename remains plain style even if matched |
| Image | no | No visual overlay or badge for matches |

### Filter Semantics Contract

| Filter | Meaning |
|--------|---------|
| All | No additional type/category restriction |
| Favorites | `isFavorite == true` |
| Links | Text/RTF items whose content is a system-openable URL |
| Text | `.text` and `.rtf` grouped together |
| Images | `.image` only |
| Files | `.file` only |

### Result-State Rules

| Condition | UI State |
|-----------|----------|
| No history exists | Show Phase 2 empty-history state |
| History exists, filtered result count > 0 | Show horizontal card list |
| History exists, filtered result count == 0, query non-empty | Show no-results state with Clear Search CTA |
| History exists, filtered result count == 0, query empty, filter active != All | Show filter-only empty helper without extra CTA |
| Search/filter pipeline error | Show search error state |

## Visual Effects

### Search UI Effects

| Element | Effect | Implementation Direction |
|---------|--------|--------------------------|
| Search field | Frosted compact material field with subtle border | SwiftUI material background + rounded rectangle stroke |
| Search focus | 1px accent ring outside field | Match system focus feel, avoid heavy glow |
| Active filter chip | Accent emphasis | Fill preferred if contrast remains accessible; otherwise stroke + tinted background |
| Match highlight | Soft rounded highlight behind substring | Use AttributedString backgroundColor or segmented Text composition |

### Animation Specifications

| Animation | Duration | Easing | Notes |
|-----------|----------|--------|-------|
| Result list update | 0.10s | easeOut | Keep fast, utility feel |
| Filter chip state change | 0.12s | easeOut | Visual confirmation only |
| Clear-search recovery | 0.10s | easeOut | Return to results quickly |
| Search focus ring transition | 0.08s | easeOut | Avoid distracting glow |

### Anti-Ambiguity Rules

- Do not animate each typed character with large transitions.
- Do not dim the entire panel during search.
- Do not add badges like “matched” to cards.
- Do not introduce second-row controls, modal search, or dropdown filter menus.

## Accessibility

### Focus and Keyboard Accessibility

| Element | Requirement |
|---------|-------------|
| Search field | Must become first keyboard focus on panel open |
| Filter chips | Must be reachable by keyboard traversal when search field loses focus |
| Card list | Regains arrow-key navigation only when search field is not editing |
| Clear Search CTA | Must be keyboard accessible and visibly focused |

### VoiceOver Labels

| Element | Accessibility Label |
|---------|---------------------|
| Search field | "Search clipboard history" |
| Filter chip | "Filter: {label}"; active state appended as "selected" |
| No-results state | "No matching results" |
| Clear Search button | "Clear search" |
| Search-highlighted card | Same base card label as Phase 2; do not announce each highlight segment separately |

### Contrast and Readability

- Highlight background must maintain legibility in both light and dark mode.
- Active chip state must achieve a clearly stronger contrast than inactive chips.
- Placeholder text must remain secondary, not low enough to become unreadable.
