  <action>
    在 `MainWindowView.swift` 将现有 `batchToolbar` 演化为 `searchableToolbar`：
    - 单行布局：搜索框在左、筛选 chips 在右，不增加第二行
    - 搜索输入使用 `TextField("Search clipboard history", text: ...)`，与 `viewModel.searchQuery` 绑定，并通过 `onChange`/setter 调用 `viewModel.setSearchQuery(_:)`
    - 筛选按钮顺序与文案固定为英文：`All`、`Favorites`、`Links`、`Text`、`Images`、`Files`
    - 点击筛选调用 `viewModel.setActiveFilter(...)`，同一时刻只有一个激活态
    - 顶栏间距与内边距遵守 UI-SPEC：水平 padding 16、上 16、下 8、搜索与筛选组间距 16、chip 间距 8
    - 搜索框高度 28，最小宽度 240，优先占据剩余空间
    - 使用 `@FocusState` 将搜索框焦点与 `viewModel.isSearchFieldFocused` 双向同步
    - 明确绑定面板“打开/重新打开”生命周期：
      - 监听 `NSWindow.didBecomeKeyNotification` 或现有面板显示回调
      - 每次面板变为 key 时调用 `viewModel.prepareForPresentation()`，确保“自动聚焦 + 重开清空”不只在 `onAppear` 生效

    增加搜索框清空 affordance（query 非空时可见）：
    - 在搜索框右侧添加 `Button` + `Image(systemName: "xmark.circle.fill")`
    - 仅当 `!viewModel.searchQuery.isEmpty` 显示
    - 点击调用 `viewModel.clearSearchQuery()` 并保持/恢复焦点（`viewModel.isSearchFieldFocused = true`）
    - 设置 `accessibilityLabel("Clear search")`，按钮样式使用 `.plain`，颜色使用 `.secondary`

    批量模式入口可保留，但必须位于顶栏最右侧或筛选组之后，且不破坏单行布局。

    如需补测试，在 `SearchFilterTests.swift` 增加断言：
    - `setActiveFilter(.favorites)` 后 `setActiveFilter(.links)` 最终为 `.links`
    - `clearSearchQuery()` 后 `searchQuery == ""`
