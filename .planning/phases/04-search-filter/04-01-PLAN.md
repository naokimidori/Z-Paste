    在 `DatabaseService.swift` 新增统一入口：
    - `func fetchMatchingItems(query: String, filter: ClipboardSearchFilter, limit: Int = 100) throws -> [ClipboardItem]`

    实现规则：
    - 先取 `fetchRecent(limit:)` 的结果（按 `created_at DESC`），再在内存中应用 `ClipboardSearchFilter.matches(item:query:)`
    - 只匹配 `content` 与文件名，不匹配 `sourceApp`
    - 保持返回顺序不变

    在 `ClipboardViewModel.swift` 新增并导出精确状态与方法：
    - `@Published var searchQuery: String = ""`
    - `@Published var activeFilter: ClipboardSearchFilter = .all`
    - `@Published var hasClipboardHistory: Bool = false`
    - `@Published var isSearchFieldFocused: Bool = false`
    - `var isFilteredEmpty: Bool { hasClipboardHistory && items.isEmpty }`
    - `func setSearchQuery(_ value: String)`：赋值并调用 `loadItems()`
    - `func setActiveFilter(_ filter: ClipboardSearchFilter)`：覆盖旧值并调用 `loadItems()`（不允许多重筛选）
    - `func clearSearchQuery()`：把 `searchQuery` 设为 "" 并调用 `loadItems()`
    - `func prepareForPresentation()`：`searchQuery = ""`、`activeFilter = .all`、`isSearchFieldFocused = true` 后调用 `loadItems()`

    修改 `loadItems()` 行为：
    - 先获取 `allItems = try database.fetchRecent(limit: 100)` 并设置 `hasClipboardHistory = !allItems.isEmpty`
    - 再用 `fetchMatchingItems(query: searchQuery, filter: activeFilter, limit: 100)` 赋值给 `items`
    - 如果 `items.isEmpty`，则 `selectedIndex = -1`；否则若旧索引越界则重置为 0
