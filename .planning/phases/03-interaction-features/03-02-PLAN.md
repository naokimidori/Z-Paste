    严格按锁定决策把菜单与 ViewModel 接通，使用以下具体接口和文案：

    1. 在 `ClipboardCardView` 中新增闭包参数：
       - `let onCopy: (() -> Void)?`
       - `let onToggleFavorite: (() -> Void)?`
       - `let onDelete: (() -> Void)?`
       并用 SwiftUI `contextMenu` 承载菜单。
    2. 菜单顺序必须写死为：
       - `Button("复制")`
       - `Button(item.isFavorite ? "取消收藏" : "收藏")`
       - `Button("删除", role: .destructive)`
       不要加入确认框、不要加入批量入口、不要加入“置顶”等 deferred idea。
    3. 在 `CardListView` 中为每张卡片同时传入 `onTap` 与三个菜单动作；右键前可先调用 `viewModel.selectItem(at: index)`，确保视觉高亮与删除后的迁移上下文一致。
    4. `onCopy` 调 `viewModel.copyItemOnly(item)`；`onToggleFavorite` 调 `viewModel.toggleFavorite(for: item)`；`onDelete` 调 `viewModel.deleteItem(item)`。
    5. `ClipboardViewModel.toggleFavorite(for:)` 和 `deleteItem(_:)` 必须在数据库成功后先做本地数组原地补丁：
       - 收藏：`items[index].isFavorite.toggle()` 或等价设置为数据库目标值
       - 删除：`items.remove(at: index)` 后调用 `reselectAfterDeletion(removedIndex: index)`
       仅在需要纠偏时再调用轻量 reload，不允许回到“每次操作都 full reload”的模式。
    6. 不要让菜单动作复用 `copySelected()`；菜单复制必须保持“不触发自动粘贴、不关闭窗口”。
  </action>
